# Workflow Rules

Status: Approved foundation rule
Owner: Saeed

## Standard Flow
1. Proposal
2. Saeed approval
3. File change
4. Review
5. Commit or push
6. Pull by the other AI environment

## Before Work
Always check the latest repository state before editing.

## During Work
Only one AI should be the main writer for a file at a time. The other AI should act as reviewer.

## After Work
Summarize what changed, why it changed, and what should happen next.

## Git Rule
Claude should pull before local work. ChatGPT should check repository state before editing when needed.

## Safety Rule
No destructive action without explicit approval.
